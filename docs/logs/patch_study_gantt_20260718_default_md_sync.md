# Study Gantt パッチ仕様書 20260718 — DEFAULT_MD のスケジュールv7同期

対象リポジトリ: `sinacost-byte/study-gantt`

---

## 1. 背景

スケジュール原本 `public/schedule.md` を v6 → v7 に内容差し替え済み（2026-07-18。B定着アークの1週後ろ倒し・C深掘りの再配分・教材再割当）。

Study Gantt の取得チェーンは「①サーバー取得 → ②localStorageキャッシュ → ③内蔵フォールバック `DEFAULT_MD`」の3段構え。仕様書の運用ルールに「**スケジュール更新時は `DEFAULT_MD` も同内容に更新する（両者の同期を保つ）**」とあるが、③はコード（`app.js` 内の定数）のため、md差し替えだけでは同期されない。現状のまま放置すると、fetch・キャッシュ双方が失敗した場面で**v6のスケジュールが⚠表示で出てくる**（内容の食い違いはフッターの⚠だけが頼りになる）。

今回の修正は仕様変更ではなく、**運用ルールに定められた同期作業＋版管理の定例作業**のみ。

## 2. 修正内容

1. `app.js` の `DEFAULT_MD` 定数の中身を、**現在の `public/schedule.md`（v7）と同一内容**に差し替える
   - 原本はリポジトリ内の `public/schedule.md` をそのまま使うこと（別の写しから貼らない。差分ゼロが受け入れ条件）
2. `app.js` の `APP_VERSION` を更新する（運用ルール:「app.js を変更するコミットでは必ず更新」。形式は現行の流儀 `YYYYMMDD-n` に従う）
3. `sw.js` の `CACHE` 定数をバンプする（運用ルール:「デプロイごとにバンプ」）

## 3. 変えないもの

- `public/schedule.md` そのもの（差し替え済み。触らない）
- パースロジック・表示ロジック一切（v7はフォーマット上v6と同一構造：メタ情報・`##`セクション・`### Week N:`・`- [ ]`/`- [x]`・マイルストーン。パーサ変更は不要）
- 取得チェーンの構造・優先順位
- localStorage のキー・進捗データ（`sg-completed`）。※タスク構成が変わった週（A Week 12・B Week 12/14・C Week 13）はIDが位置ベースでずれるが、いずれも未着手の週のため実害なし。移行処理は書かないこと
- ファイル名（`public/schedule.md` のリネームは禁止。7/05障害の再発防止）

## 4. README 更新

- 「実装判断で決めた値」等に変更があれば追記（今回はおそらく無し）。変更履歴に「20260718: DEFAULT_MD をスケジュールv7に同期、APP_VERSION/CACHE更新」の1行を追加

## 5. 受け入れ基準（AKが端末で確認）

1. デプロイ後、**2回リロード**してフッターの app 版スタンプが新 `APP_VERSION` になっている（SWワンリロード遅延の考慮）
2. フッターの版表示が `版: 1Q 〜2026-08-29 / 取得元: サーバー取得` である（通常経路の健全性）
3. ガントチャートに v7 の変更が反映されている（目視ポイント: B「Week 12: 定着① ネットワーク再走＋TW実地観察」が7/25まで伸びている／C「Week 13: 深掘り・外 — MCPサーバ実践」が7/26〜8/15／マイルストーンに8/15が2件・8/16・8/29が2件）
4. 既存の進捗チェック（Week 1〜11相当）が保持されている
5. **フォールバック確認**: 機内モード等でネットワークを切り、localStorageのキャッシュ（`sg-schedule-md`）を削除した状態で起動 → ⚠付き「内蔵フォールバック」表示になり、**その内容がv7である**こと（今回の修正の本丸）。確認後は通常状態に戻して 2. を再確認
