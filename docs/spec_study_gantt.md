# 学習管理アプリ（Study Gantt） — 仕様書

## 概要

| 項目 | 内容 |
|------|------|
| **アプリ名** | Study Gantt |
| **URL** | `sinacost-byte.github.io/study-gantt/` |
| **目的** | 「Self-Study with AI University」の学習スケジュールを俯瞰し、進捗を管理する |
| **ユーザー** | AK（1人。個人学習用） |
| **端末** | iPhone / iPad（PWA）メイン、PC（MacBook Air）でも閲覧可 |
| **設計思想** | ガントチャート一本でシンプルに。「見る」と「完了をつける」だけ。書き込まない、いじらない |

---

## 背景

- AKは普段から仕事でガントチャート（Brabio）を使っており、スケジュールの俯瞰・進捗把握にガントチャートを好む
- 本プロジェクトは3Quarter制（1Q: 4-6月、2Q: 7-9月、3Q: 10-12月）で、1Qに3科目（A/B/C）を並行して進める
- シラバスは Markdown ファイルで管理されており、Claude と相談しながら更新する運用
- スケジュールの「原本」は Markdown。アプリはそれを表示・可視化する読み取り専用のビューワー

---

## コンセプト：Single Source of Truth

```
Markdown（シラバス） ──読み込み──→ アプリ（表示＋進捗管理）
       ↑                              ↓
  Claude と相談して修正          進捗データはアプリ側に保存
```

- **スケジュール・タスクの定義**: Markdown が唯一の原本。アプリ側では編集しない
- **進捗データ（完了/未完了）**: アプリ側に保存する（mdには書き戻さない）
- Markdown を修正して再読み込みすると、スケジュールが更新される。進捗データは保持される

---

## 機能

### コア機能

1. **ガントチャート表示**: 科目×週のスケジュールをガントチャートで表示する
2. **表示スパン切り替え**: 週 / 月 / 全体 の3段階で表示範囲を切り替えられる
3. **期間ナビゲーション**: ◀ ▶ ボタンで週/月単位でスクロール。「今日」ボタンで現在週/月に戻る
4. **進捗管理（タスク単位）**: 各タスクのチェックボックスをタップして「完了」をつける
5. **進捗管理（週単位・自動）**: その週のタスクがすべて完了すると、週のバーが自動で「完了」表示になる（「完了」バッジ付き）
6. **進捗管理（科目単位・自動）**: 科目内の全タスクが完了すると、科目バーにも「完了」バッジが付く
7. **マイルストーン表示**: 大きなゴール（Quarter終了、統合レポート等）と到達点（各フェーズの区切り等）を◆マーカーで表示する
8. **「今日」の表示**: ガントチャート上に赤い縦線で「今日」の位置を表示し、現在地が一目で分かる
9. **折りたたみ**: 左列の科目⇒週⇒タスクを3段階で折りたたみ/展開できる
10. **Markdown 読み込み**: 📁ボタンから新しいスケジュールmdファイルを読み込んで更新できる
11. **ラベルツールチップ**: 見切れたラベルの全文を確認できる（下記参照）

### ラベルツールチップ

左列のラベルは幅の制約で見切れることがある。全文を確認する操作：

| 行タイプ | 操作 | 動作 |
|---------|------|------|
| task | ラベル部分をタップ | 全文ツールチップ表示 |
| week | ラベル部分を長押し（500ms） | 全文ツールチップ表示 |
| section | — | そのまま表示（短文のため見切れない） |

- ツールチップ外タップで閉じる
- 長押し検知後は折りたたみを発火しない（フラグ制御）
- iOS長押し時のコンテキストメニューは `-webkit-touch-callout: none` で抑止

### つけない機能（意図的に省略）

- スケジュール編集（アプリ側ではしない。md を修正して再読み込み）
- 詳細なタスク記入・メモ・ファイル添付
- タスクの詳細画面（タップして詳細に飛ぶ、は不要）
- カレンダー表示（ガントチャートの表示スパン切り替えで代替）
- 通知・リマインダー（Apple 純正リマインダーの役割）
- 他ユーザーとの共有・コラボレーション
- Google カレンダー連携

## 画面構成

### メイン画面（ガントチャート）

```
┌──────────────────────────────────────────────────┐
│  [週] [月] [全体]        ← 表示スパン切り替え     │
│  ◀ 3/29 − 4/4 ▶         ← 期間ナビゲーション     │
│  [📅今日]    [📁md読込]   ← 操作ボタン             │
├──────────────────────────────────────────────────┤
│  1Q              │3/29(日) 3/30(月) 3/31(火)  … │
│  ─────────────────┼──────────────────────────────│
│  ▼ オリエンテーション 完了│ ████████████████████   │
│    ☑ カリキュラム全体… │ ████████████████████     │
│    ☑ CS50x翻訳プレ…   │ ████████████████████     │
│    …                   │                          │
│  ─────────────────┼──────────────────────────────│
│  ▼ A: 技術革命史  │           ████████████████   │
│    ▶ Week 1: 技術… │          ████               │
│    ▶ Week 2: 活版… │               ████          │
│  ─────────────────┼──────────────────────────────│
│  ▶ B: CS入門      │           ████████████████   │
│  ▶ C: AI/ML基礎   │           ████████████████   │
│  ─────────────────┼──────────────────────────────│
│  ★ マイルストーン  │      ◆              ◆       │
│                    │              ↓今日           │
├──────────────────────────────────────────────────┤
│  Self-Study with AI University     6/9 完了      │
└──────────────────────────────────────────────────┘
```

- 左列：科目名 → 週タイトル → タスク（3段階折りたたみ）
  - ▼/▶ で展開・折りたたみをトグル
  - 初期状態：科目は展開、週は折りたたみ（科目バーと週バーが見える状態）
- 右側：時間軸にバーを表示
- 表示スパンに応じてバーの粒度が変わる：
  - **週表示**: 日単位（曜日付き）。その週のタスクが詳細に見える
  - **月表示**: 週単位。1ヶ月の流れが見える
  - **全体表示**: 月単位。1Q〜3Q の全体が俯瞰できる
- フッター：アプリ名と完了タスク数（例：「6/9 完了」）

### タスク完了の操作

- チェックボックスをタップすると「完了/未完了」がトグルする
- バー上をタップしても完了をトグルできる
- 完了 = チェックボックスが塗りつぶされ、バーの色が淡色に、チェックマーク付き。ラベルに取り消し線
- 誤タップした場合はもう一度タップで戻せる

## デザイン

### テーマ

- **明るいテーマ（白背景）** で固定。ダークモード非対応

### 科目カラー（実装値）

| 要素 | メインカラー | ライト | ライター | テキスト |
|------|------------|--------|---------|---------|
| オリエンテーション | `#78716C` | `#E7E5E4` | `#F5F5F4` | `#44403C` |
| A: 技術革命史 | `#D97706` | `#FDE68A` | `#FEF3C7` | `#92400E` |
| B: CS入門 | `#2563EB` | `#93C5FD` | `#DBEAFE` | `#1E3A8A` |
| C: AI/ML基礎 | `#0D9488` | `#5EEAD4` | `#CCFBF1` | `#134E4A` |
| マイルストーン | `#DC2626` | — | — | — |
| 「今日」の線 | `#DC2626` | — | — | — |

- 科目バー（section）: メインカラーで塗りつぶし、白文字でラベル表示
- 週バー（week）: ライトカラー
- タスクバー: メインカラーの半透明（`55`）＋ボーダー
- 完了済み: ライターカラー＋ライトカラーのボーダー

### レイアウト

- 左列ラベル幅: 180px 固定
- 行の高さ: section 40px / week 36px / task 32px / milestone 32px
- フォント: `-apple-system, BlinkMacSystemFont, "Hiragino Sans"` 系

## データ構成

### 初期データ

- **1Q のスケジュール md をアプリのソースコードに埋め込み**（`DEFAULT_MD` 定数）
- 初回起動時：埋め込みの md を自動でパースし、すぐにガントチャートが表示される
- 更新時：📁ボタンからファイル選択で新しい md を読み込み直す
- 読み込み後、確認ダイアログ:「スケジュールを更新しますか？（進捗データは保持されます）」

### 入力：Markdown フォーマット

```markdown
# 1Q スケジュール

## メタ情報
- 開始日: 2026-03-28
- 終了日: 2026-06-27
- Quarter: 1Q
- 週の開始曜日: 日曜日

---

## オリエンテーション（2026-03-28 〜 2026-04-04）
- [ ] カリキュラム全体の確認・学習環境セットアップ
- [ ] CS50x翻訳プレーヤーの動作確認

## A: 技術革命史

### Week 1: 技術革命の全体地図（2026-04-05 〜 2026-04-11）
- [ ] 講義: 技術革命のリスト、「革命」の定義、技術決定論 vs 社会構成主義
- [ ] 読書:『第四次産業革命』シュワブ
- [ ] ディスカッション: AI革命の歴史的位置づけ

## マイルストーン
- 2026-04-04: オリエンテーション完了
- 2026-04-25: A科目パターン抽出①（Week 1-3）

（以下、B科目・C科目も同様の形式）
```

**パースルール**:
- `## セクション名` → 科目グループ（オリエンテーション、A、B、C、マイルストーン）
- `### Week N: タイトル（YYYY-MM-DD 〜 YYYY-MM-DD）` → 週。括弧内の日付でバーを配置
- `## セクション名（YYYY-MM-DD 〜 YYYY-MM-DD）` → 日付付きセクション（オリエンテーション等）
- `- [ ] テキスト` → 未完了タスク
- `- [x] テキスト` → 完了済みタスク（md上での初期状態指定用）
- `## マイルストーン` 配下の `- YYYY-MM-DD: ラベル` → マイルストーン
- `## メタ情報` 配下 → Quarter全体の設定情報

**タスクIDの生成ルール**:
- `{Quarter}-{科目}-{Week}-task{N}` の形式で自動生成
- 例：`1Q-A-W1-task1`、`1Q-B-W3-task2`、`1Q-ORI-task1`（オリエンテーション）
- md を再読み込みしたとき、同じ位置のタスクは同じIDが振られ、進捗データが保持される

### 出力（進捗データの保存）

- 進捗データ（どのタスクが完了済みか）は `window.storage`（Claude artifact persistent storage）に保存
- ストレージキー: `sg-completed`
- データ形式:
  ```json
  {
    "1Q-ORI-task1": true,
    "1Q-ORI-task2": true,
    "1Q-A-W1-task1": true
  }
  ```
- Markdown を再読み込みしても、タスクIDが一致する限り進捗データは保持される
- ブラウザキャッシュクリア時にはデータが失われる（既知の制約）

## 日付処理の注意事項

- 日付文字列のパース: `new Date(s + 'T00:00:00')` で時刻をローカル深夜0時に正規化
- `getWeekStart()`: 週の開始日（日曜）を取得する際に `setHours(0,0,0,0)` で時刻を正規化する（`navDate` が現在時刻を持つため、正規化しないと `dayDiff` のround で1日ズレる）
- `dayDiff()`: `Math.round((b - a) / 86400000)` で日数差を計算
