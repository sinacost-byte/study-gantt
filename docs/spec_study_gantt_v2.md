# 学習管理アプリ（ガントチャート） — 仕様書

## 概要

| 項目 | 内容 |
|------|------|
| **アプリ名** | 未定（仮：Study Gantt） |
| **目的** | 「Self-Study with AI University」の学習スケジュールを俯瞰し、進捗を管理する |
| **ユーザー** | AK（1人。個人学習用） |
| **端末** | iPad / iPhone（PWA） |
| **設計思想** | ガントチャート一本でシンプルに。「見る」と「完了をつける」だけ。書き込まない、いじらない |

---

## 背景

- AKは普段から仕事でガントチャート（Brabio）を使っており、スケジュールの俯瞰・進捗把握にガントチャートを好む
- 本プロジェクトは3Quarter制（1Q: 4-6月、2Q: 7-9月、3Q: 10-12月）で、1Qに3科目（A/B/C）を並行して進める
- シラバスは Markdown ファイルで管理されており、Claude と相談しながら更新する運用
- スケジュールの「原本」は Markdown。アプリはそれを表示・可視化する読み取り専用のビューワー

---

## コンセプト：Single Source of Truth

```
Markdown（シラバス） ──読み込み──→ アプリ（表示＋進捗管理）
       ↑                              ↓
  Claude と相談して修正          進捗データはアプリ側に保存
```

- **スケジュール・タスクの定義**: Markdown が唯一の原本。アプリ側では編集しない
- **進捗データ（完了/未完了）**: アプリ側に保存する（mdには書き戻さない）
- Markdown を修正して再読み込みすると、スケジュールが更新される。進捗データは保持される

---

## 機能

### コア機能

1. **Markdown 読み込み**: シラバスの Markdown ファイルを読み込み、ガントチャートのデータに変換する
2. **ガントチャート表示**: 科目×週のスケジュールをガントチャートで表示する
3. **表示スパン切り替え**: 週 / 月 / 年（全体）の3段階で表示範囲を切り替えられる
4. **進捗管理（タスク単位）**: 各タスクをタップして「完了」をつける。シールをぺたっと貼る感覚
5. **進捗管理（週単位・自動）**: その週のタスクがすべて完了すると、週のバーが自動で「完了」表示になる
6. **マイルストーン表示**: 大きなゴール（Quarter終了、統合レポート等）と小さな到達点（各フェーズの区切り等）を表示する
7. **「今日」の表示**: ガントチャート上に「今日」の線を表示し、現在地が一目で分かるようにする
8. **折りたたみ**: 左列の科目⇒週⇒タスクを3段階で折りたたみ/展開できる

### つけない機能（意図的に省略）

- スケジュール編集（アプリ側ではしない。md を修正して再読み込み）
- 詳細なタスク記入・メモ・ファイル添付
- タスクの詳細画面（タップして詳細に飛ぶ、は不要）
- カレンダー表示（ガントチャートの表示スパン切り替えで代替）
- 通知・リマインダー（Apple 純正リマインダーの役割）
- 他ユーザーとの共有・コラボレーション
- Google カレンダー連携（初版では不要）

## 画面構成

### メイン画面（ガントチャート）

```
┌──────────────────────────────────────────────────┐
│  [週] [月] [年]          ← 表示スパン切り替え     │
│  ◀ ▶                    ← 期間ナビゲーション      │
│  [📁 md読み込み]          ← ファイル選択ボタン     │
├──────────────────────────────────────────────────┤
│                    │4/5   4/12  4/19  4/26  5/3  │
│  ─────────────────┼──────────────────────────────│
│  ▼ A: 技術革命史   │ ████  ████  ████             │
│    ▼ Week 1       │ ████                          │
│      □ 講義       │ ██                            │
│      □ 読書       │   ██                          │
│      □ ディス…    │    ██                         │
│    ▶ Week 2       │       ████                    │
│    ▶ Week 3       │             ████              │
│  ─────────────────┼──────────────────────────────│
│  ▶ B: CS入門      │ ████  ████  ████             │
│  ─────────────────┼──────────────────────────────│
│  ▶ C: AI/ML基礎   │ ████  ████  ████             │
│  ─────────────────┼──────────────────────────────│
│  ★ マイルストーン  │             ◆ パターン分析①  │
│                    │                      ↓今日   │
└──────────────────────────────────────────────────┘
```

- 左列：科目名 → 週タイトル → タスク（3段階折りたたみ）
  - ▼/▶ で展開・折りたたみをトグル
  - 初期状態：科目は展開、週は折りたたみ（科目バーと週バーが見える状態）
- 右側：時間軸にバーを表示
- 表示スパンに応じてバーの粒度が変わる：
  - **週表示**: 日単位。その週のタスクが詳細に見える
  - **月表示**: 週単位。1ヶ月の流れが見える
  - **年表示**: 月単位。1Q〜3Q の全体が俯瞰できる
- タスクバーをタップ → 完了トグル（色が変わる）
- 完了した週のバーは自動で色が変わる

### タスク完了の操作

- タスクバーをタップすると「完了/未完了」がトグルする
- 完了 = バーの色が薄くなり、チェックマークが付く
- 操作は指1本、タップ1回
- 誤タップした場合はもう一度タップで戻せる

## デザイン

### テーマ

- **明るいテーマ（白背景）** で固定。ダークモード非対応（初版）

| 要素 | カラー | 用途 |

| オリエンテーション | グレー系（#9E9E9E） | 準備期間。控えめ |

- タップ領域は最低 44x44pt（Apple HIG準拠）

## データ構成

### 初期データ

- **1Q のスケジュール md をアプリに同梱する**（別ファイルとして `schedule.md` をリポジトリに配置）
- 初回起動時：同梱の md を自動で読み込み、すぐにガントチャートが表示される
- 更新時：ファイル選択（📁ボタン）から新しい md を読み込み直す
- 同梱 md はフォールバック（データが飛んだ場合のリセット用）も兼ねる

### 入力：Markdown フォーマット

アプリが読み込む Markdown の形式を定義する。既存のシラバス md から必要な情報を抽出しやすい構造にする。

```markdown
# 1Q スケジュール

## メタ情報
- 開始日: 2026-03-28
- 終了日: 2026-06-27
- Quarter: 1Q
- 週の開始曜日: 日曜日

---

## オリエンテーション（2026-03-28 〜 2026-04-04）
- [ ] カリキュラム全体の確認・学習環境セットアップ
- [ ] CS50x翻訳プレーヤーの動作確認
- [ ] 学習管理アプリ（ガントチャート）の動作確認
- [ ] Python環境構築（ローカル or Google Colab）
- [ ] 予習読書:『技術革新と不平等の1000年史（上）』読了
- [ ] 予習読書:『ネット興亡記』読了

## A: 技術革命史

### Week 1: 技術革命の全体地図（2026-04-05 〜 2026-04-11）

- [ ] 読書:『第四次産業革命』シュワブ（薄い本。全体の地図として）
- [ ] ディスカッション: AI革命の歴史的位置づけ

### Week 2: 活版印刷 — 知識の民主化（2026-04-12 〜 2026-04-18）
- [ ] 講義: グーテンベルク、知識の独占の崩壊
- [ ] 読書:『グーテンベルクの銀河系』マクルーハン
- [ ] ディスカッション:「じわじわ」の変化を深掘りする

## マイルストーン
- 2026-04-04: オリエンテーション完了
- 2026-04-25: A科目パターン抽出①（Week 1-3）
- 2026-06-27: 1Q 統合レポート提出（A/B/C 全科目）

（以下、B科目・C科目も同様の形式）
```

**パースルール**:
- `## セクション名` → 科目グループ（オリエンテーション、A、B、C、マイルストーン）
- `### Week N: タイトル（YYYY-MM-DD 〜 YYYY-MM-DD）` → 週。括弧内の日付でバーを配置
- `## セクション名（YYYY-MM-DD 〜 YYYY-MM-DD）` → 日付付きセクション（オリエンテーション等）
- `- [ ] テキスト` → 未完了タスク
- `- [x] テキスト` → 完了済みタスク（md上での初期状態指定用）
- `## マイルストーン` 配下の `- YYYY-MM-DD: ラベル` → マイルストーン
- `## メタ情報` 配下 → Quarter全体の設定情報

**タスクIDの生成ルール**:
- `{Quarter}-{科目}-{Week}-task{N}` の形式で自動生成
- 例：`1Q-A-W1-task1`、`1Q-B-W3-task2`、`1Q-ORI-task1`（オリエンテーション）
- md を再読み込みしたとき、同じ位置のタスクは同じIDが振られ、進捗データが保持される

### 出力（進捗データの保存）

- 進捗データ（どのタスクが完了済みか）はアプリのローカルストレージに保存する
- データ形式例:
  ```json
  {
    "completed_tasks": [
      "1Q-ORI-task1",
      "1Q-ORI-task2",
      "1Q-A-W1-task1",
      "1Q-A-W1-task2",
      "1Q-B-W1-task1"
    ],
    "last_updated": "2026-04-10T20:30:00"
  }
  ```
- Markdown を再読み込みしても、タスクIDが一致する限り進捗データは保持される

## Markdown 読み込みの運用フロー

```
1. Claude とシラバスを相談・更新する
2. シラバス md からアプリ用 md を作成（Claude に依頼可能）
3. アプリの📁ボタンからファイル選択で読み込み
4. ガントチャートが更新される
5. 進捗データは保持される（タスクIDが変わらない限り）
```

### 読み込み方法

- **ファイル選択**: 端末の「ファイル」アプリから md ファイルを選択してアップロード
- 画面上部に「📁 md読み込み」ボタンを配置
- 読み込み後、確認ダイアログ:「スケジュールを更新しますか？（進捗データは保持されます）」
- 読み込み成功後、ガントチャートが即座に更新される
